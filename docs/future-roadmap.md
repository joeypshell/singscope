# Future roadmap

Highest-value work after physical-iPhone acceptance:

1. Run the hardware matrix, record measured pitch/display latency and rendering FPS, then tune queue/frame thresholds per route without changing raw data.
2. Move to a dedicated isolated domain with CSP and Permissions-Policy response headers.
3. Add calibrated input/output latency measurement and optional guide-tone scheduling.
4. Improve long-source streaming analysis and worker-generated multiresolution review tiles.
5. Add richer, accessible piano-roll editing, note splitting/merging, tempo-map display, and MusicXML import only after threat/size review.
6. Add encrypted user-controlled backup destinations. No cloud/account dependency is planned without an explicit product decision.
7. Evaluate a native iPhone wrapper only if PWA audio-route or storage constraints remain release blockers.

Source separation, lyric transcription, social features, subscriptions, streaming integrations, opaque AI scores, and medical/vocal-health claims remain outside the MVP.
